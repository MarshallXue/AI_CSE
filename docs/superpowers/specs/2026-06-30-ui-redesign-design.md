# 岸岸通 UI 改版设计

## 背景

本设计基于前期错题与闪卡复盘规格，以及 UI 灵感调研资料夹 `ui-inspiration/`。

目标不是重做产品，而是在现有移动端 PWA 上做第一轮视觉和核心体验升级，让岸岸通从“可用原型”提升到更清爽、更可信、更有复盘记忆点的学习工具。

## 设计方向

总方向：

> 清爽学习工具 + 纸面阅读感 + 沉浸闪卡

具体拆分：

- 今日页和错题库：清爽学习工具感。
- 新闻详情和解析内容：明显纸面阅读感，但正文保持现代清爽。
- 复盘页和闪卡页：沉浸式单卡复习感。

粉笔只参考功能心智，不参考视觉风格。可借鉴其公考模块分类、题目解析层级和练习结果表达，但不采用传统题库 App 的拥挤视觉、课程售卖感和土蓝土绿配色。

## 视觉基调

### 色彩

主色定为：

- 墨蓝：主按钮、导航选中态、标题重点、复盘主入口。
- 琥珀：考点高亮、今日重点、`需巩固` 状态、挖空答案。
- 绿色：`已掌握` 和完成状态。
- 低饱和红：`未掌握`。
- 背景：近白和极浅灰蓝。
- 卡片：白底、轻边框、柔和阴影。

避免：

- 粉笔式亮蓝亮绿。
- AI 紫蓝渐变。
- 大面积毛玻璃。
- 高饱和彩虹色。

### 卡片

默认组合：

- 今日页和错题库：轻边框卡片，圆角 10-12px。
- 新闻详情和解析：略带纸张感的阅读卡片。
- 闪卡页：克制分层大卡片，圆角约 16px。

闪卡分层只表达“还有复习队列”，不做夸张 3D、旋转堆叠或大阴影。

### 字体与排版

- 中文字体使用系统字体：`PingFang SC / Microsoft YaHei / system-ui`。
- 标题使用更有力的 Semibold/Bold。
- 正文保持高可读性。
- 新闻正文和解析正文行高建议 1.6-1.7。
- 新闻详情要有明显纸面感：标题、导语、关键词和考点层级更清楚，但正文仍使用现代无衬线。

### 动效

动效只服务学习状态和操作反馈：

- 卡片翻面 150-220ms。
- 按钮按压反馈。
- 完成一张卡后的轻微推进。
- 列表筛选和底部抽屉淡入淡出。

不做粒子、强 3D、复杂滚动叙事或游戏化爆炸反馈。

## 第一轮改版范围

第一轮做五个核心 UI 模块。

### 1. 底部导航

导航顺序调整为：

1. 今日
2. 复盘
3. 错题库
4. 我的

目标：

- 强化“复盘是每日核心动作”。
- 保持移动端原生工具感。
- 选中态使用墨蓝，未选中态弱化。

要求：

- 图标和文字都保留。
- 不做纯图标导航。
- 触摸区域清晰。

### 2. 复盘首页

复盘首页是闪卡复习中心，不是错题列表和学习报告页。

页面层级：

1. 今日复盘概览
   - 今日待复盘总数。
   - 预计时间。
   - 今日完成进度。
2. 主按钮
   - `开始今日全部复盘`。
3. 三个入口卡
   - 错题闪卡。
   - 词汇闪卡。
   - 时政考点闪卡。
4. 轻量成就感
   - 连续复盘天数或本周完成张数。
   - 不做强打卡、勋章或游戏化体系。

每个入口卡展示：

- 今日到期。
- 逾期。
- 预计时间。
- 完成进度。

### 3. 闪卡页

闪卡页是第一轮最重要的体验。

视觉：

- 克制分层卡片。
- 中央主卡最大，背后露出 1-2 层浅色卡片边缘。
- 顶部只保留进度和必要退出入口。
- 底部固定操作区。

交互：

- 正面展示题目、词语或挖空。
- 错题支持 `我已作答` 和 `直接看解析`。
- 翻面后展示答案、解析、关键词或释义。
- 翻面后出现三档反馈：`未掌握 / 需巩固 / 已掌握`。

状态按钮：

- 未掌握：淡红底 + 深红字。
- 需巩固：淡琥珀底 + 棕黄字。
- 已掌握：淡绿底 + 深绿字。

列表和详情中状态标签弱化，闪卡背面状态按钮强化。

### 4. 今日词汇组卡

词汇数据使用现有 `apps/frontend/src/app/data/vocabulary-entries.json`。

词汇组卡展示：

- 组名。
- 类型/来源，例如成语、实词、上册/下册。
- 词条数。
- 3-5 个词语预览。
- 学习状态：未开始 / 已打开 / 已进入复盘。

点开词汇组后：

- 展示组内词条。
- 每个词展示释义和例句。
- 该组自动进入词汇复习池。
- 默认状态为 `需巩固`。

视觉要求：

- 借鉴不背单词的语境感，但不做壁纸化和强情绪氛围。
- 词语预览让用户一眼知道本组内容。

### 5. 错题导入与识别结果页

错题库导入入口中等突出：

- 顶部右侧主按钮：`导入错题`。
- 下方三个小入口：拍照、上传图片、上传 PDF。
- 不做大面积上传区，错题库仍以模块管理为主。

导入流程：

1. 用户选择拍照、图片或 PDF。
2. PDF 上传后按页预览，支持全选。
3. 开始识别。
4. 展示 OCR / 切题 / 生成解析 / 分类进度。
5. 识别完成后进入结果页。

结果页规则：

- 拍一题时可以直接进入错题详情。
- 多题或 PDF 导入时展示识别结果列表。
- 用户勾选要保存的题目。
- 顶部提供全选。
- 顶部显示已选数量，例如 `已选 8/12`。
- 底部固定操作栏：
  - 保存所选到错题库。
  - 保存并加入复盘。

默认：

- 普通识别结果默认选中。
- 建议检查的题也默认选中，但显示提醒标签。
- 用户可以取消勾选。
- 没有选中题目时，底部按钮置灰。

## 手机端错题编辑

移动端编辑不能做成电脑式整页表单。

原则：

> 默认展示清楚，编辑按块进入，尽量选择代替输入。

错题详情阅读态：

1. 题目图片缩略区
   - 保留原图。
   - 可点开查看大图。
   - 可显示来源页码和切题位置。
2. 结构化题目区
   - 题干。
   - 选项。
   - 正确答案。
   - 用户答案。
3. AI 解析区
   - 解题思路。
   - 正确项说明。
   - 易错项说明。
   - 错因分析。
4. 标签区
   - 模块。
   - 知识点。
   - 错因。
   - 状态。
5. 底部操作
   - 加入复盘。
   - 重新识别。
   - 保存。

编辑方式：

- 点题干区 `编辑`，打开底部抽屉，只编辑题干。
- 点选项区 `编辑`，打开选项编辑抽屉。
- 点答案区 `编辑`，优先弹出选择器，不要求打字。
- 点模块/知识点，使用选择器或标签搜索。
- 点错因，使用常用错因 chips。
- 点解析，使用独立编辑抽屉或编辑页。

编辑入口要轻但可见。区块标题右侧放小型 `编辑` 即可，不要每行都放图标。

## 新闻详情和时政考点

新闻详情采用明显纸面感，但正文保持现代清爽。

结构：

- 新闻标题。
- 时间、来源、主题标签。
- 热点或高频考点标识。
- 导语浅底纸张块。
- 正文分段阅读。
- 关键词轻高亮。
- AI 考点区。

时政考点展示方式：

- 显示核心考点列表。
- 点开某个考点后查看生成的挖空卡。
- 考点卡默认进入时政复习池，状态为 `需巩固`。

不在新闻详情里直接铺满所有挖空卡，避免干扰阅读。

## 成功标准

第一轮 UI 改版完成后应满足：

- 底部导航顺序符合产品主轴：今日 -> 复盘 -> 错题库 -> 我的。
- 复盘首页一眼看出“开始闪卡复习”，而不是错题列表。
- 闪卡页有明显产品记忆点：克制分层卡片、单卡聚焦、三档反馈。
- 今日词汇组比普通列表更有内容感，能看到 3-5 个词语预览。
- 错题导入流程适合手机端，用户能勾选保存，支持全选。
- 错题编辑不是整页表单，而是阅读态详情加分块底部抽屉。
- 整体视觉不再像传统题库 App，也不走花哨教育 App 路线。
