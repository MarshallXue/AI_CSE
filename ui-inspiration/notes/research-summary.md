# UI 调研汇总

## 设计读法

岸岸通不是花哨教育 App，也不是传统课程售卖 App。它更像一个公考备考者每天愿意打开的学习桌面：

- 今日板块像一份可学习的公考晨报。
- 错题库像可管理的错题资产库。
- 复盘板块像沉浸式闪卡训练中心。

推荐组合：

> 粉笔的可信工具感 + 不背单词的沉浸卡片 + Quizlet/Anki 的单卡复习流 + Awwwards 的阅读排版节奏

## 值得借鉴

### 粉笔

适合借鉴：

- 公考模块组织方式。
- 题目解析页的信息层级。
- 练习/复盘结果里的数据表达。
- 工具型学习产品的可信感。

不适合照搬：

- 密集课程/营销入口。
- 传统题库首页的拥挤感。
- 过多考试商业化信息。

### 不背单词

适合借鉴：

- 大卡片沉浸学习。
- 真实语境例句。
- 重点词义和关键词高亮。
- 轻量复习仪式感。

不适合照搬：

- 大面积壁纸和情绪氛围。
- 过强潮流感或娱乐感。
- 英语学习 App 的语境包装方式。

### Anki / Quizlet

适合借鉴：

- 顶部进度。
- 中央单卡。
- 翻面后再反馈。
- 复习过程减少干扰。

不适合照搬：

- Anki 原生设置复杂度。
- 过于工程化的卡组管理心智。

### 扇贝 / 墨墨

适合借鉴：

- 今日任务和抗遗忘规划。
- 复习数量可控。
- 长期记忆曲线的产品表达。

不适合照搬：

- 过重的背词体系术语。
- 为英语学习服务的复杂词库路径。

## 页面升级建议

### 今日页

目标：从信息列表升级为今日学习桌面。

建议：

- 顶部显示今日日期、时政数量、词汇组数、待复盘数量。
- 新闻卡片做轻报纸感，标题清楚，来源/时间/主题弱化。
- 重要新闻使用小型 `热点` 或 `高频考点` 标识。
- 今日词汇用词汇组卡片展示，强调“点开后进入复习池”。

### 错题库

目标：从列表仓库升级为可管理错题资产。

建议：

- 顶部保留拍照、上传图片、上传 PDF。
- 模块入口使用规整模块卡。
- 模块卡显示总错题、待复盘、需巩固数量。
- 错题状态标签弱化显示。
- 筛选使用横向 chip，不做复杂表单。

### 复盘页

目标：成为产品最有记忆点的核心体验。

建议：

- 首页突出 `开始今日全部复盘`。
- 三个入口：错题闪卡、词汇闪卡、时政考点闪卡。
- 每个入口显示到期数量、逾期数量、预计时间。
- 闪卡页顶部只保留进度。
- 中央大卡片，底部固定反馈按钮。
- 翻卡动效轻，150-220ms，避免夸张 3D。

### 词汇卡

目标：不做普通词表，做公考中文词汇的真实语境记忆。

建议：

- 词义卡：词语、释义、易混、例句。
- 语境卡：例句挖空、完整句子、关键词加粗。
- 词语大字号，释义行距舒适。
- 挖空处使用浅色高亮块。

## 视觉原则

- 主色：沉稳青蓝或靛青蓝。
- 辅色：淡青绿，用于完成和词汇相关内容。
- 强调色：暖琥珀，用于重点考点、热点和挖空答案。
- 未掌握：低饱和红。
- 需巩固：淡琥珀。
- 已掌握：淡绿。
- 背景：近白色和极浅灰蓝。
- 圆角：整体控制在 8px 左右，避免过度圆润。
- 阴影：少用大阴影，多用浅边框、浅底色和层级留白。
- 动效：服务状态变化和学习反馈，不做炫技动画。

## 反向约束

不要做：

- AI 紫蓝渐变。
- 大面积毛玻璃。
- 游戏化金币、宠物、过强签到。
- Dribbble 式彩虹渐变和 3D 卡片堆。
- Awwwards 式复杂滚动叙事。
- 传统题库 App 的满屏拥挤。
- 小屏后台 Dashboard 感。
